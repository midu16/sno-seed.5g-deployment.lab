# Contributing to SNO Seed Cluster

Thank you for your interest in contributing to the SNO Seed cluster deployment project!

## Getting Started

1. Fork the repository
2. Clone your fork locally
3. Create a feature branch from `main`
4. Make your changes
5. Test your changes thoroughly
6. Submit a pull request

## Code Standards

- Follow existing code style and patterns
- Document any configuration changes
- Test deployment changes in a lab environment before submitting
- Keep commits atomic and well-described

## Pull Request Process

1. Update the README.md with details of changes to the deployment process
2. Update any relevant documentation
3. The PR will be merged once reviewed and approved

## Reporting Issues

- Use the GitHub issue tracker
- Provide detailed information about your environment
- Include relevant logs and error messages
- Describe steps to reproduce the issue

## Questions?

Feel free to open an issue for questions or discussions about the deployment process.
